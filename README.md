# 수시 입시 결과 분석 프로그램

대교협 대입상담프로그램에서 내려받은 엑셀 파일을 업로드하면, 브라우저 안에서 곧바로 통계와 시각화를 보여주는 단일 HTML 페이지 프로그램입니다.

- **2026 강원진학센터 입시분석팀** 제작 (담당: 남궁연 · 설악고)
- 서버 전송 없이 **브라우저 안에서만** 계산합니다. 개인정보가 외부로 나가지 않습니다.
- 파일 하나(`index.html`)만 있으면 어디서든 실행할 수 있습니다.

## 빠른 시작

1. 이 저장소의 [`index.html`](./index.html)을 내려받습니다.
   - GitHub 페이지의 **Code → Download ZIP** 또는 `git clone`
   - 또는 [Releases](https://github.com/ngryun/susi_for_GWE/releases)에 올라온 최신 파일
2. 내려받은 `index.html`을 **더블클릭**해서 브라우저로 엽니다.
   - Chrome / Edge / Safari / Firefox 최신 버전 권장
3. **엑셀 업로드** 영역에서 입시 결과 파일(`.xlsx`, `.xls`)을 선택합니다. 여러 해의 파일을 한 번에 올려도 됩니다.
4. **보고서 생성** 버튼을 누르면 잠시 후 분석 화면이 나타납니다.

> 💡 **온라인으로 바로 쓰기**: 단순한 정적 HTML이라서 [GitHub Pages](https://pages.github.com/), [Netlify](https://www.netlify.com/), [Cloudflare Pages](https://pages.cloudflare.com/) 같은 무료 정적 호스팅에 그대로 올려도 동작합니다.

## 입력 데이터 양식

대교협 **대입상담프로그램 → 진학관리**에서 받은 엑셀을 그대로 올리면 자동으로 인식합니다.

| 다운로드 위치 | 비고 |
|---|---|
| 수시진학관리 → 엑셀다운로드 | 학생별 환산등급·수능정보까지 포함 (권장) |
| 전년도 수시진학 → 엑셀다운로드 | 환산등급·수능정보가 누락될 수 있음 |

- 여러 해의 파일을 함께 업로드하면 **연도별 비교**가 활성화됩니다.
- 학교(단위학교) 양식과 기존 통합본 양식 모두 지원합니다 — 헤더 행으로 자동 판별합니다.
- 학생 식별자(학년·반·번호 또는 학생 고유코드)는 익명 키로 변환되어 분석에 사용됩니다.

## 분석 페이지

업로드 후 상단 탭으로 3개 페이지를 전환할 수 있습니다.

1. **전체데이터요약** — 전형별 현황, 지역별 현황, 등급대 차트, 지원 상위 대학 TOP 20, (다년치 업로드 시) 연도별 비교
2. **조건별조회** — 대학·학과·전형 세부유형을 검색·다중 선택하고, 카테고리끼리는 AND 조건으로 묶어 특정 대학의 특정 세부유형처럼 좁혀서 확인
3. **지원경향조회** — 학생 단위로 어떤 등급대가 어디에 지원했고 합격/불합격이 어떻게 갈렸는지 한 줄로 보기

각 차트·표에서는 다음 기능을 사용할 수 있습니다:

- 막대·셀 클릭으로 **상세 데이터 모달** 열기 (대학명/세부유형/모집단위/결과 필터 지원)
- 차트 우상단 메뉴에서 **PNG / SVG 저장**, 일부 차트는 **표로 보기**
- 연도별 비교의 두 표(전형별 현황·주요 대학 지원 추이)는 **엑셀 저장** 버튼 제공
- 상세 모달에서 **현재 필터 기준으로 엑셀 저장**

## 결과 공유

분석 결과 화면 상단의 **공유** 메뉴에서:

- **현재 페이지만** 또는 **전체 보고서**를 단일 HTML 파일로 저장
- **암호 보호** 옵션 — 받은 사람이 비밀번호를 입력해야 데이터가 복호화됩니다 (AES-GCM + PBKDF2)

저장된 HTML은 받은 사람의 브라우저에서 그대로 열어보면 분석 결과가 재현됩니다.

## 개인정보 보호

- 업로드한 엑셀은 **외부 서버로 전송되지 않습니다.** 모든 파싱·집계가 브라우저(JavaScript) 안에서 처리됩니다.
- 페이지를 닫으면 데이터는 자동으로 사라집니다 — 별도 캐시·쿠키에 저장되지 않습니다.
- 학교명·학생 식별 정보는 업로드 직후 폐기되거나 익명 키로 변환됩니다.

## 기술 스택

- 순수 HTML + CSS + JavaScript (프레임워크 없음)
- [SheetJS (xlsx)](https://sheetjs.com/) — 엑셀 파싱 (배포용 HTML에 포함)
- [Plotly.js](https://plotly.com/javascript/) — 차트 (배포용 HTML에 포함)
- Web Crypto API — 공유 파일 암호화

> 배포용 `index.html`에는 엑셀 파서와 차트 라이브러리가 함께 포함되어 있어 CDN 접속 없이도 실행됩니다.

## 개발자 가이드

배포물은 `index.html` 단일 파일이지만, 개발은 `src/` 디렉터리의 분리된 소스에서 진행하고 한 파일로 다시 합치는 방식입니다.

```
src/
├── index.template.html   # HTML 뼈대
├── styles.css            # 전체 스타일
├── app.js                # 메인 앱 로직 (모든 분석·UI)
└── vendor/               # 단일 HTML에 포함할 외부 라이브러리
tools/
└── single-file.js        # 빌드 / 추출 / 검증 스크립트
index.html                # 배포용 단일 파일 (위 셋을 합친 결과)
```

### 빌드 명령어

Node.js만 설치되어 있으면 됩니다.

```bash
# src/ → index.html 다시 만들기
node tools/single-file.js build

# 현재 index.html을 src/ 로 다시 분해
node tools/single-file.js extract

# src/ 와 index.html이 일치하는지 확인
node tools/single-file.js check
```

### 작업 흐름

1. `src/app.js`, `src/styles.css`, `src/index.template.html`을 수정합니다.
2. `node tools/single-file.js build`로 `index.html`을 재생성합니다.
3. 브라우저로 `index.html`을 열어 동작을 확인합니다.
4. 커밋에는 `src/`의 수정본과 빌드된 `index.html`을 함께 포함합니다.

## 호환 환경

- 데스크탑 브라우저: Chrome 90+, Edge 90+, Firefox 90+, Safari 15+
- 모바일 브라우저에서도 동작하지만 화면이 좁아 데스크탑 사용을 권장
- 인터넷 연결: 프로그램 실행과 엑셀 분석에는 필요 없음
